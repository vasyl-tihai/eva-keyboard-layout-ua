# Пропозиції покращення розкладок (дані, не смак)

Цей документ — результат кількісного аудиту розкладок репозиторію.
Усі цифри пораховані скриптом [analyzer.py](analyzer.py) на корпусі з ~250 000 літер
(випадкові статті української Вікіпедії) з **чесною моделлю пальців**:

- внутрішня колонка натискається **тим самим вказівним пальцем** (у першій версії
  оптимізатора вона вважалась окремим «пальцем», через що частина реальних SFB не рахувалась);
- біграми рахуються **лише всередині слів** — між словами реальний набір іде через
  пробіл на великому пальці.

> ⚠️ Вікіпедія ≠ ваші чати. Перед тим як міняти розкладку під себе, перерахуйте
> метрики на власних текстах: `python analyzer.py мій_корпус.txt`.

## Діагноз Еви 3.8

| Метрика | Значення | Коментар |
|---|---|---|
| SFB | **3.29 %** | добре проти ЙЦУКЕН (18.1 %), але є резерв ×2 |
| Чергування рук | 57.8 % | добре |
| Домашній ряд | **57.8 %** | найкращий показник серед усіх розкладок у таблиці |
| Мізинці | **17.4 %** | багато: И (5.3 %) на лівому мізинці + Е (4.8 %) на правому |

Топ SFB-біграм: **кр** 0.38 %, **чн** 0.27 %, **рс** 0.19 %, **ск** 0.19 %, **др** 0.13 %, **бр** 0.12 %.

Майже всі вони — навколо **лівого вказівного пальця** (він накриває К, С, Б, Д, Р, М),
плюс пара **чн** на правому вказівному.

**Ключове спостереження:** у кожній із цих пар щонайменше одна літера стоїть ПОЗА
домашнім рядом (к, ч, д, б — верхній/нижній ряди). Отже, конфлікти можна розвести,
взагалі не чіпаючи домашній ряд — філософія «основні літери на домашньому ряду»
залишається недоторканою.

## 🥇 Пропозиція «Ева 3.9» — три обміни поза домашнім рядом (рекомендую)

Обміняти **М ↔ Ч**, **К ↔ Ц**, **Д ↔ З** (усі шість літер — на верхньому/нижньому рядах;
домашній ряд, мізинці й голосний блок не зачеплені):

```
[   ] [ Я ] [ Й ] [ П ] [ Ц ] [ З ]        [ Г ] [ Л ] [ У ] [ І ] [ Щ ] [ Ї ]
[ Є ] [ И ] [ В ] [ Т ] [ С ] [ Р ]        [ Х ] [ Н ] [ О ] [ А ] [ Е ] [ Ю ]
[   ] [ Ж ] [ К ] [ Д ] [ Б ] [ Ч ]        [ Ш ] [ М ] [ , ] [ . ] [ Ь ] [ Ф ]
```

| Метрика | Ева 3.8 | Ева 3.9 |
|---|---|---|
| SFB | 3.29 % | **1.89 %** (−42 %) |
| Домашній ряд | 57.8 % | **57.8 %** (без змін) |
| Мізинці | 17.4 % | 17.4 % (без змін) |
| Перекати до вказівного | 17.0 % | **18.5 %** |
| Чергування рук | 57.8 % | 56.5 % |

Чому це працює:

- **М ↔ Ч**: Ч тікає з правого вказівного від Н — «чн» (0.27 %) стає чергуванням рук.
- **К ↔ Ц**: К сходить з колонки вказівного — зникають «кр» (0.38 %), «ск»/«кс» (0.29 %).
- **Д ↔ З**: Д відходить від Р — зникають «др»/«рд» (0.19 %).

Бонус для перевчання: домашній ряд — найглибша м'язова пам'ять — не змінюється взагалі.
Якщо хочеться зовсім мінімального кроку — сам лише обмін **М ↔ Ч** дає 3.29 % → 2.83 %.

Залишок SFB майже весь — «рс» (0.19 %): Р і С сусідять на домашньому ряду вказівного
пальця. Прибрати його можна лише виселивши одну з них з домашнього ряду
(наприклад, обміном Й ↔ Р: SFB впаде до ~1.7 %, але домашній ряд просяде на 4 п.п.) —
це вже питання філософії, і в межах принципу «домашній ряд понад усе» я б залишив як є.

## Пропозиція «Ева 3.9+» — п'ять обмінів, мінімальний SFB (експериментальна)

Якщо SFB важливіший за домашній ряд: обміни **Й↔Р**, **М↔Ч**, **Г↔Й**, **А↔У**, **Е↔У**:

```
[   ] [ Я ] [ Р ] [ П ] [ К ] [ Д ]        [ Й ] [ Л ] [ А ] [ І ] [ Щ ] [ Ї ]
[ Є ] [ И ] [ В ] [ Т ] [ С ] [ Г ]        [ Х ] [ Н ] [ О ] [ Е ] [ У ] [ Ю ]
[   ] [ Ж ] [ Ц ] [ З ] [ Б ] [ Ч ]        [ Ш ] [ М ] [ , ] [ . ] [ Ь ] [ Ф ]
```

- **SFB 1.71 %**, мізинці 17.4 % → **16.1 %** (Е йде з мізинця).
- Ціна: А (8.4 % — друга за частотою літера) на верхньому ряду, домашній ряд
  просідає до 48.8 %. На колонково-ступінчастій клавіатурі верхній ряд середнього
  пальця — позиція майже домашня, але це вже свідомий відхід від принципу
  «основні літери на домашньому ряду».

## Пропозиція «Ліна 2.0» — перегенерувати чесною моделлю

Стара Ліна оптимізувалась кодом із двома вадами (окремий «палець» на внутрішній
колонці + міжслівні біграми), тому вона платить за красиве чергування рук
**24.8 % навантаження на мізинці** — зокрема Е (4.8 %) стоїть на верхньому ряду лівого мізинця.

Виправлений `main.py` (500 000 ітерацій × 4 рестарти, seed 42, той самий корпус) дає:

```
[ Ї ] [ В ] [ Л ] [ Д ] [ Х ]        [ Ь ] [ И ] [ А ] [ П ] [ Ц ] [ Є ]
[ С ] [ Р ] [ Н ] [ Т ] [ Ч ]        [ У ] [ І ] [ О ] [ К ] [ З ] [ Ж ]
[ Ю ] [ Й ] [ М ] [ Г ] [ Щ ]        [ Ґ ] [ Е ] [ Я ] [ Б ] [ Ф ] [ Ш ]
```

- **SFB 0.48 %** (удвічі менше за стару Ліну), мізинці **11.3 %** (замість 24.8 %),
  перекати до вказівного 16.3 % (замість 11.1 %).
- Цікаво: виправлений алгоритм **сам** відтворив головний принцип Еви — приголосні
  зліва, голосні (И-А-У-І-О-Е) праворуч. Ручна інтуїція і чесна математика зійшлися.

## Canary Phonetic UA — чесна ціна фонетичності

SFB 4.65 % — найвищий серед кастомних розкладок, і майже весь він у колонці
середнього пальця лівої руки **И-С-В**: біграми **ви** (0.70 %), **ис**, **ив**, **си**, **вс**
разом дають ~2 % SFB. Це прямий наслідок фонетичного принципу (И на Y, С на S, В на V) —
виправити, не зламавши фонетику, неможливо. Пропозиція: залишити як є, але чесно
написати в README, що Canary Phonetic UA — це свідомий обмін ~1.5 п.п. SFB на
нульове перевчання, і для «повністю українських» днів краще пересідати на Еву.

## Дрібні пропозиції

1. **Задокументувати Ґ і апостроф.** У матрицях Еви їх не видно: Ґ живе на утриманні Г,
   апостроф — на комбо. В українських текстах апостроф трапляється частіше за Ґ
   (0.11 % проти 0.01 % на вікі-корпусі), тож він вартий власного рядка в README.
2. **Узгодити Eva.vil зі схемою.** У Vial-матриці файла Ї/Ю/Ф підключені в четвертому
   ряду, конфіг містить 46 активних позицій, PrtScr/Tab/Delete на нульовому шарі —
   README має чесно описувати, для якої плати цей файл, або варто зібрати «чистий»
   варіант під стандартний Corne 42.
3. **Опублікувати Canary Phonetic UA як файл** (.vil або keymap.json), а не лише картинку.

## Як відтворити

```bash
# метрики всіх розкладок на своєму корпусі
python analyzer.py мій_корпус.txt

# згенерувати власну «Ліну» (чесна модель, хвилини замість діб)
python main.py --corpus corpus_texts --iterations 500000 --restarts 4 --seed 42
```
