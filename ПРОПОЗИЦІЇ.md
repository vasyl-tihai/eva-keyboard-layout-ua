# Пропозиції покращення розкладок (дані, не смак)

Цей документ — результат кількісного аудиту розкладок репозиторію.
Усі цифри пораховані скриптом [analyzer.py](analyzer.py) на **змішаному корпусі з
6.2 млн літер** трьох жанрів:

| Піджанр | Джерело | Обсяг |
|---|---|---|
| Збалансований письмовий (преса, художня, наука) | [Brown-UK](https://github.com/brown-uk/corpus) | ~3.0 млн літер |
| Розмовний (діалоги) | [OpenSubtitles UA](https://opus.nlpl.eu/OpenSubtitles.php) | ~3.0 млн літер |
| Енциклопедичний | випадкові статті [Вікіпедії](https://uk.wikipedia.org/) | ~0.25 млн літер |

Модель пальців чесна: внутрішня колонка натискається **тим самим вказівним
пальцем**, біграми рахуються **лише всередині слів** (між словами — Space на
великому пальці).

> ⚠️ Перед перевчанням перерахуйте метрики на власних текстах:
> `python analyzer.py мій_корпус.txt`. Висновки нижче перевірені на трьох жанрах
> окремо — де жанр впливає на результат, це зазначено.

## Діагноз Еви 3.8 (змішаний корпус)

| Метрика | Значення | Коментар |
|---|---|---|
| SFB | **2.92 %** | у 6 разів краще за ЙЦУКЕН (18.3 %), але є резерв |
| Чергування рук | 57.1 % | добре |
| Домашній ряд | **58.3 %** | найкращий показник серед усіх розкладок у таблиці |
| Мізинці | **19.9 %** | багато: И (6.2 %) на лівому мізинці + Е (5.6 %) на правому |

Топ SFB-біграм: **кр** 0.27 %, **ск** 0.21 %, **чн** 0.20 %, **жи** 0.16 %, **бр** 0.16 %, **др** 0.12 %.

Майже всі вони — навколо **лівого вказівного пальця** (він накриває К, С, Б, Д, Р, М),
плюс пара **чн** на правому вказівному.

**Ключове спостереження:** у кожній із цих пар щонайменше одна літера стоїть ПОЗА
домашнім рядом (к, ч, д, б — верхній/нижній ряди). Отже, конфлікти можна розвести,
взагалі не чіпаючи домашній ряд — філософія «основні літери на домашньому ряду»
залишається недоторканою.

## 🥇 Пропозиція «Ева 3.9» — три обміни поза домашнім рядом (рекомендую)

> ✅ **Прийнято.** Ева 3.9 додана в репозиторій ([Eva-3.9.vil](Eva-3.9.vil)) і стала
> рекомендованою версією в README.

Обміняти **М ↔ Ч**, **К ↔ Ц**, **Д ↔ З** (усі шість літер — на верхньому/нижньому рядах;
домашній ряд, мізинці й голосний блок не зачеплені):

```
[   ] [ Я ] [ Й ] [ П ] [ Ц ] [ З ]        [ Г ] [ Л ] [ У ] [ І ] [ Щ ] [ Ї ]
[ Є ] [ И ] [ В ] [ Т ] [ С ] [ Р ]        [ Х ] [ Н ] [ О ] [ А ] [ Е ] [ Ю ]
[   ] [ Ж ] [ К ] [ Д ] [ Б ] [ Ч ]        [ Ш ] [ М ] [ , ] [ . ] [ Ь ] [ Ф ]
```

| Метрика | Ева 3.8 | Ева 3.9 |
|---|---|---|
| SFB (змішаний корпус) | 2.92 % | **1.96 %** (−33 %) |
| Домашній ряд | 58.3 % | **58.3 %** (без змін) |
| Мізинці | 19.9 % | 19.9 % (без змін) |
| Перекати до вказівного | 17.0 % | **17.8 %** |
| Чергування рук | 57.1 % | 55.4 % |

Стійкість до жанру (SFB, Ева 3.8 → Ева 3.9):

| Жанр | 3.8 | 3.9 | Виграш |
|---|---|---|---|
| Письмовий (Brown-UK) | 2.95 % | 1.86 % | −37 % |
| Розмовний (субтитри) | 3.04 % | 2.22 % | −27 % |
| Вікіпедія | 3.29 % | 1.89 % | −42 % |

Чому це працює:

- **М ↔ Ч**: Ч тікає з правого вказівного від Н — «чн» стає чергуванням рук.
- **К ↔ Ц**: К сходить з колонки вказівного — зникають «кр», «ск»/«кс».
- **Д ↔ З**: Д відходить від Р — зникають «др»/«рд».

Бонус для перевчання: домашній ряд — найглибша м'язова пам'ять — не змінюється взагалі.
Якщо хочеться зовсім мінімального кроку — сам лише обмін **М ↔ Ч** дає 2.92 % → 2.67 %.

Нюанси, які лишаються:

- **«мн»** (0.10 % загалом, але 0.21 % у розмовному стилі — слово «мені»): М після обміну
  сусідить із Н на правому вказівному. Якщо друкуєте переважно чати, додатковий обмін
  **Б ↔ М** прибирає «мн» (розмовний SFB 2.22 % → 2.14 %), але в письмових жанрах він
  трохи шкодить — раджу спершу пожити з 3.9 і подивитись, чи «мн» взагалі відчувається.
- **«рс»** (0.07 %): Р і С сусідять на домашньому ряду вказівного. Прибрати можна лише
  виселивши одну з них з домашнього ряду — в межах принципу «домашній ряд понад усе»
  не варто.

## Пропозиція «Ева 3.9+» — п'ять обмінів, мінімальний SFB (експериментальна)

Обміни **Й↔Р**, **М↔Ч**, **Г↔Й**, **А↔У**, **Е↔У**:

```
[   ] [ Я ] [ Р ] [ П ] [ К ] [ Д ]        [ Й ] [ Л ] [ А ] [ І ] [ Щ ] [ Ї ]
[ Є ] [ И ] [ В ] [ Т ] [ С ] [ Г ]        [ Х ] [ Н ] [ О ] [ Е ] [ У ] [ Ю ]
[   ] [ Ж ] [ Ц ] [ З ] [ Б ] [ Ч ]        [ Ш ] [ М ] [ , ] [ . ] [ Ь ] [ Ф ]
```

На змішаному корпусі: SFB 1.87 %, мізинці 17.7 %, але домашній ряд просідає до 50.5 %.
Перевага над «Евою 3.9» у SFB — лише 0.09 п.п., а ціна — 8 п.п. домашнього ряду і
9 переставлених літер замість 6. **З урахуванням великого корпусу цей варіант втратив
сенс — «Ева 3.9» краща майже за всім.** Залишаю для історії.

## Пропозиція «Ліна 2.0» — перегенерувати чесною моделлю

Стара Ліна оптимізувалась кодом із двома вадами (окремий «палець» на внутрішній
колонці + міжслівні біграми), тому вона платить за красиве чергування рук
**24.5 % навантаження на мізинці**.

Виправлений `main.py` (500 000 ітерацій × 4 рестарти, seed 42, змішаний корпус) дає:

```
[ Ю ] [ З ] [ Л ] [ Д ] [ Ц ]        [ Ь ] [ Е ] [ А ] [ Р ] [ Г ] [ Ф ]
[ К ] [ С ] [ Н ] [ Т ] [ Б ]        [ У ] [ И ] [ О ] [ В ] [ П ] [ Х ]
[ Ї ] [ Ч ] [ Є ] [ М ] [ Щ ]        [ Ґ ] [ І ] [ Я ] [ Й ] [ Ш ] [ Ж ]
```

- **SFB 0.53 %**, мізинці ~12 % (замість 24.5 %), перекати до вказівного 15.8 %.
- Головне — **стійкість до жанру**. Версія, згенерована лише на Вікіпедії, на
  розмовному тексті деградувала до 1.66 % SFB; ця тримає 0.47–0.80 % в усіх трьох жанрах:

| Жанр | Ліна 2.0 (змішана) | Ліна на самій вікі |
|---|---|---|
| Письмовий | 0.47 % | 0.51 % |
| Розмовний | **0.80 %** | **1.66 %** |
| Вікіпедія | 0.51 % | 0.48 % |

Мораль: **оптимізуйте на тому, що реально друкуєте**, і обов'язково змішуйте жанри —
однорідний корпус дає розкладку, що ламається на іншому стилі тексту.

## Canary Phonetic UA — чесна ціна фонетичності

SFB 5.15 % — найвищий серед кастомних розкладок, і майже весь він у колонці
середнього пальця лівої руки **И-С-В**: біграми «ви», «ис», «ив», «си», «вс» разом
дають ~2 % SFB. Це прямий наслідок фонетичного принципу (И на Y, С на S, В на V) —
виправити, не зламавши фонетику, неможливо. Пропозиція: залишити як є, але чесно
позиціонувати: Canary Phonetic UA — це свідомий обмін ~2 п.п. SFB на нульове
перевчання для тих, хто здебільшого друкує англійською.

## Дрібні пропозиції

1. **Задокументувати Ґ і апостроф.** У матрицях Еви їх не видно: Ґ живе на утриманні Г,
   апостроф — на комбо. Апостроф в українських текстах трапляється частіше за Ґ,
   тож він вартий власного рядка в README.
2. **Узгодити Eva.vil зі схемою.** У Vial-матриці файла Ї/Ю/Ф підключені в четвертому
   ряду, конфіг містить 46 активних позицій, PrtScr/Tab/Delete на нульовому шарі —
   README має чесно описувати, для якої плати цей файл, або варто зібрати «чистий»
   варіант під стандартний Corne 42.
3. **Опублікувати Canary Phonetic UA як файл** (.vil або keymap.json), а не лише картинку.

## Як відтворити

```bash
# метрики всіх розкладок на своєму корпусі
python analyzer.py мій_корпус.txt

# згенерувати власну «Ліну» (чесна модель, хвилини замість діб)
python main.py --corpus corpus_texts --iterations 500000 --restarts 4 --seed 42
```

Корпуси, використані тут: [Brown-UK](https://github.com/brown-uk/corpus) (git clone),
[OpenSubtitles UA mono](https://object.pouta.csc.fi/OPUS-OpenSubtitles/v2018/mono/uk.txt.gz),
випадкові статті Вікіпедії через [API](https://uk.wikipedia.org/w/api.php).
